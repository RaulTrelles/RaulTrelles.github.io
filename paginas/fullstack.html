Sistema Talonario

El Proyecto

Talonario, es un sistema concebido para apoyar a las instituciones en su necesidad de ayudar en la tarea de fiscalización a los ciudadanos, el control a los administrados y/o apoyar en la gestión de las infracciones o controles sobre los administrados.





Implementación



Como desarrollador back-end y front-end, fui responsable de construir toda la interfaz de usuario para el sitio web, Aplicativo Móbil y las Api's que forman parte de la solución, definir la experiencia del usuario y las interacciones entre el usuario y la interfaz más aya de la comodidad en su uso, fueron el reto del presente proyecto.




Una de las experiencias más gratificantes fue integrar todo el sistema front-end con el back-End haciendo uso de Flutter, NodeJS, PostGreSql haciendo uso de los beneficios de AWS, Vercel y Cloudinary.

Portal
Permite visualizar las fiscalizaciones realizadas en línea, así como configurar nuevos usaurios, Fiscalizaciones y seguimientos
Login
Se evalua accesos y funciones a reaizar de acuerdo a sus accesos
Móvil
Aplicativo Capturando la información o evidencia de la infracción
Back-end
Portal desarrolladoe en NodeJs, haciendo uso de express, swagger, cors, pg "manejo de PostGreSql", variables de entorno, cloudinary, multer, fs, pdfkits, pdfkits-tablet entre otras.

Sistema Courrier

El Proyecto

Currier, es una solución que recepciona documentación y productos que seran luego distribuidos como una empresa courrier, dicho esto y luego de generar la orden de reparto asignada a un despacho, este hace seguimiento a sus entregas por medio de una app, la cual con la dirección tiene una ubicación geográfica de todos las antregas a realizar..





Implementación



Como desarrollador back-end y front-end, fui responsable de generar el backend y la app, para el personal de hacer las entregas de los documentos, para ello integré la app a Google map así como generé las api's necesarias para que el aplicativo tuviera la información necesaria para poder hacer la entrega de los repartos.




Una de las experiencias más gratificantes fue integrar todo el sistema front-end con el back-End haciendo uso de Flutter, NodeJS, SQL Server haciendo uso de los beneficios de AWS, Vercel.

Sistema WebScraping Seace

El Proyecto

WebScreaping, es una solución que descarga del portal de contrataciones del estado, todo lo oblicado por este, el mismo que es analizado por temas y entidades de interes para luego generar excel y notificaciones que permiten hacer un seguimiento a los temas de interes del interesado..





Implementación



Como desarrollador back-end y front-end, fui responsable de generar el backend haciendo uso de FastApi incorporanto Selenium como libreria de WebScraping lo que permitio organizar la información que es almacenada en una base de datos SQL Server, para ser luego consumida mediante el back-end en el portal de analisis desarrollado en ReFlex usando Python como lenguaje de desarrollo integral.




La estrategia del desarrollo fue desarrollar la solución integralmente en Python usando tecnologias como FastApi como Back-end, Reflex como Front.end predomianando una componentización de la solución.
